## Assignment Overview

This repository contains my Product Manager assignment for designing a basic e-commerce system MVP from scratch.

The proposed MVP focuses on completing the core transaction flow:

**Browse Products → Add to Cart / Buy Now → Submit Order → Pay → Ship → Complete Order**

The solution is designed from a Product Manager perspective, covering product scope, user roles, feature prioritization, business process design, technical understanding, exception handling, and launch metrics.

---

## Product Objective

The goal of this MVP is to launch a reliable first version of an e-commerce system that allows customers to complete purchases while enabling internal staff to manage products, inventory, orders, and shipment operations.

The design prioritizes:

- End-to-end transaction reliability
- Clear MVP scope control
- Accurate inventory and order status handling
- Practical admin/operations workflows
- Basic technical understanding for working with engineering teams
- Measurable launch performance through key product metrics

---

## Contents Covered

The submitted document includes the following sections:

1. **User Roles and Core Scenarios**  
   Defines the main users of the system: Customer/Buyer, Operations Staff, and Platform Admin.

2. **MVP Feature List and Priority**  
   Prioritizes features using P0, P1, and P2 classification based on business necessity and transaction flow impact.

3. **Core Process Design**  
   Details the customer order/payment process and product creation/publishing process, including normal flows, exceptions, and status changes.

4. **Simplified PRD**  
   Provides a simplified Product Requirements Document for the Product Detail Page.

5. **API Understanding Answers**  
   Explains how product detail API fields should be interpreted and displayed on the frontend.

6. **Code Logic Reading Answers**  
   Translates simple order-submission logic into business rules and identifies missing validation scenarios.

7. **Order Status Understanding Answers**  
   Describes the order lifecycle and status transitions from pending payment to completion, cancellation, or refund.

8. **Bug / Exception Analysis Answers**  
   Analyzes possible causes of inventory inconsistency between frontend display and backend validation.

9. **Data Metrics Design**  
   Defines key launch metrics such as product detail page views, add-to-cart rate, order conversion rate, payment success rate, GMV, fulfillment time, and refund rate.

## Product Thinking Highlight

This assignment is not only a feature list. It also considers practical product and technical risks, including:

- Backend validation as the final source of truth
- SKU-level inventory control
- Product off-sale handling
- Payment timeout and stock release
- Price changes before checkout
- Order status machine design
- Frontend/backend data inconsistency
- Operational efficiency after launch
